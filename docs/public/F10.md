## 概述

F10 模块提供通达信 F10 公司基本面资料数据的获取和解析功能。

**模块位置**: 源码编译 (`--features f10`)，pip 包不包含此模块

**连接方式**: 独立连接 (不占用共享连接池)

**数据来源**: 通达信服务器 `get_company_info_category` / `get_company_info_content` API

### F10 数据分类 (600519 贵州茅台)

| 分类 | 大小 | 分类 | 大小 |
|------|------|------|------|
| 最新提示 | 12KB | 公司概况 | 13KB |
| 财务分析 | 36KB | 股东研究 | 25KB |
| 股本结构 | 4KB | 资本运作 | 13KB |
| 业内点评 | 70KB | 行业分析 | 16KB |
| 公司大事 | 357KB | 研究报告 | 70KB |
| 经营分析 | 23KB | 主力追踪 | 28KB |
| 分红扩股 | 47KB | 高层治理 | 19KB |
| 龙虎榜单 | 25KB | 关联个股 | 23KB |

全量约 **780KB** / **267K 字符**。

## 快速开始

### 安装 (从源码编译)

```bash
git clone https://github.com/jiangtaovan/tdxrs && cd tdxrs
pip install maturin
maturin develop --release --features f10
```

### 使用

```python
from tdxrs.pro import TdxF10Client

# 创建客户端 (独立连接，指定服务器)
client = TdxF10Client("180.153.18.170", 7709)

# 获取分类列表
categories = client.get_category(1, "600519")
for cat in categories:
    print(f"{cat['name']}: {cat['length']} bytes")

# 获取公司概况 (推荐: 传入分类字典)
content = client.get_content(1, "600519", categories[1])
print(content[:500])

# 解析为结构化数据
parsed = TdxF10Client.parse_f10(content)
print(f"公司名称: {parsed['basic_info'].get('公司名称')}")
```

## API 参考

### TdxF10Client

F10 公司资料客户端，使用独立连接，不占用共享连接池。

```python
from tdxrs.pro import TdxF10Client

client = TdxF10Client("180.153.18.170", 7709)          # 默认超时 10s
client = TdxF10Client("180.153.18.170", 7709, timeout=15.0)  # 自定义超时
```

#### 连接管理

| 方法 | 返回 | 说明 |
|------|------|------|
| `TdxF10Client(ip, port, timeout=10.0)` | — | 创建客户端 |
| `set_server(ip, port)` | — | 切换服务器 |
| `set_timeout(secs)` | — | 设置超时时间 |

---

#### get_category

```python
get_category(market: int, code: str) -> list[dict]
get_category_auto(code: str) -> list[dict]
```

获取 F10 分类列表。每个分类包含名称、文件名、起始位置和长度。

| 参数 | 类型 | 说明 |
|------|------|------|
| `market` | int | 市场代码 (0=SZ, 1=SH) |
| `code` | str | 股票代码 |

返回字段: `name`, `filename`, `start`, `length`

```python
categories = client.get_category(1, "600519")
# 或自动识别市场
categories = client.get_category_auto("600519")
```

---

#### get_content

```python
get_content(market: int, code: str, category: dict) -> str
```

获取指定分类的文本内容。**推荐使用此方法**，传入 `get_category` 返回的字典。

| 参数 | 类型 | 说明 |
|------|------|------|
| `market` | int | 市场代码 |
| `code` | str | 股票代码 |
| `category` | dict | 分类字典 (从 get_category 获取) |

```python
categories = client.get_category(1, "600519")

# 获取公司概况
content = client.get_content(1, "600519", categories[1])
print(content[:500])

# 获取财务分析
fin_content = client.get_content(1, "600519", categories[2])
```

---

#### get_all_contents / get_all_data

```python
get_all_contents(market: int, code: str) -> dict[str, str]
get_all_data(market: int, code: str) -> dict
```

获取所有分类的内容。`get_all_data` 返回结构化数据。

```python
# 获取所有内容 (字典)
all_contents = client.get_all_contents(1, "600519")
print(f"获取 {len(all_contents)} 个分类")

# 获取结构化数据
data = client.get_all_data(1, "600519")
print(f"分类数: {data['category_count']}")
print(f"总字符: {data['total_chars']}")
print(f"总字节: {data['total_bytes']}")
```

`get_all_data` 返回字段:

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | str | 股票代码 |
| `market` | int | 市场代码 |
| `category_count` | int | 分类数量 |
| `total_chars` | int | 总字符数 |
| `total_bytes` | int | 总字节数 |
| `contents` | list[dict] | 各分类内容 (`category`, `content`, `byte_length`) |

## 数据格式

### 原始文本格式

F10 数据使用港澳资讯格式，以 `【数字.标题】` 标记章节：

```
公司概况  ◇600519 贵州茅台  更新日期：2026-06-19◇  通达信沪深京F10
★本栏包括【1.基本资料】【2.发行和交易】【3.员工效益】【4.研发投入】【5.参股控股】

【1.基本资料】
┌───────┬───────────────────────────────┐
│公司名称      │贵州茅台酒股份有限公司                                        │
├───────┼───────────────────────────────┤
│证券代码      │600519                │
└───────┴───────────────────────────────┘
```

### 表格解析

Rust 解析器支持以下表格格式：

- `│字段名│值│` (全角竖线 U+2502)
- `｜字段名｜值｜` (全角竖线 U+FF5C)
- `|字段名|值|` (半角竖线 U+007C)

## 最佳实践

### 批量获取多只股票

```python
from tdxrs.pro import TdxF10Client

client = TdxF10Client("180.153.18.170", 7709)

stocks = [
    (1, "600519"),  # 贵州茅台
    (1, "601318"),  # 中国平安
    (0, "000858"),  # 五粮液
]

for market, code in stocks:
    categories = client.get_category(market, code)
    content = client.get_content(market, code, categories[1])  # 公司概况
    parsed = TdxF10Client.parse_f10(content)
    name = parsed['basic_info'].get('公司名称', 'N/A')
    print(f"{code}: {name}")
```

### 本地缓存

```python
import json, os
from datetime import datetime, timedelta

class F10Cache:
    def __init__(self, cache_dir="f10_cache", ttl_hours=24):
        self.cache_dir = cache_dir
        self.ttl = timedelta(hours=ttl_hours)
        os.makedirs(cache_dir, exist_ok=True)

    def get(self, code):
        path = os.path.join(self.cache_dir, f"{code}.json")
        if os.path.exists(path):
            mtime = datetime.fromtimestamp(os.path.getmtime(path))
            if datetime.now() - mtime < self.ttl:
                with open(path, 'r', encoding='utf-8') as f:
                    return json.load(f)
        return None

    def set(self, code, data):
        path = os.path.join(self.cache_dir, f"{code}.json")
        with open(path, 'w', encoding='utf-8') as f:
            json.dump(data, f, ensure_ascii=False, indent=2)

cache = F10Cache()
data = cache.get("600519")
if not data:
    client = TdxF10Client("180.153.18.170", 7709)
    categories = client.get_category(1, "600519")
    content = client.get_content(1, "600519", categories[1])
    data = TdxF10Client.parse_f10(content)
    cache.set("600519", data)
```

## 模块结构

```
src/profile/
├── mod.rs              # 模块入口, 重导出
├── constants.rs        # 协议常量、分类名称
├── types.rs            # F10 数据类型 (F10Category, F10Content, F10Data)
├── parser.rs           # 二进制响应解析器 (分类列表 + 内容)
├── parser_f10.rs       # F10 文本解析器 (结构化提取)
└── client.rs           # F10 客户端 (共享连接池变体)

src/net/
└── f10_client.rs       # F10 客户端 (独立连接, Python 绑定使用)
```

### 设计原则

1. **独立连接**: 不占用共享连接池，避免影响行情数据
2. **原始字节保留**: `F10Category` 保留 GBK 原始字节，避免编码损耗
3. **内置解析器**: Rust 实现 F10 文本解析，267K 字符仅需 4.6ms
4. **兼容 tdxpy**: 数据格式与 tdxpy 的 F10Parser 兼容

## 常见问题

### Q: F10 客户端为什么使用独立连接？

A: F10 数据包体较大 (单分类 5KB-357KB，全量约 780KB)，全量获取可能持续数秒。使用独立连接避免占用共享连接池，不影响行情数据的实时性。

### Q: 推荐用 get_content 还是 get_content_by_name？

A: 推荐 `get_content(dict)`。直接传入 `get_category` 返回的字典，避免分类名称的编码往返问题。

### Q: 不同公司的 F10 数据格式有差异吗？

A: 有轻微差异。分类名称可能不完全相同，但基本格式一致。Rust 解析器已做兼容处理。

### Q: F10 数据多久更新一次？

A: 通常每天或每周更新一次，具体取决于数据提供商。
